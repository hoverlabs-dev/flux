# Portal - Maya Blender Asset Bridge (FBX)

Portal is a standalone PyQt6 launchpad designed for moving selected mesh assets seamlessly between Autodesk Maya and Blender using FBX, while preserving critical data: UV layouts, materials, transforms, pivots, hierarchy, sharp edges, and custom properties.

---

## 🚀 Quick Start

1. **Launch Portal**: Run the bridge launchpad.
   - Double-click `Launch Portal.bat`
   - Or run `python run_portal.py` in your terminal.

2. **Configure Settings**:
   - Go to the **Settings** tab.
   - Set the file paths to your **Maya Executable** (`maya.exe`) and **Blender Executable** (`blender.exe`).
   - Set your preferred **Exchange Directory** (where FBX assets and manifest data will be stored temporarily).
   - Click **Save Settings**.

3. **Launch DCCs**:
   - From the Portal launchpad, click **Launch Maya** and/or **Launch Blender**.
   - This opens the applications and bootstraps their local background servers to listen to Portal.

4. **Synchronize Meshes**:
   - **Maya ➔ Blender**:
     - Select one or more meshes in Maya.
     - Set the direction switcher in Portal to `Maya ➔ Blender`.
     - Click **ROUNDTRIP SYNC** (or click **Manual Export** in Maya and then **Manual Import** in Blender).
   - **Blender ➔ Maya**:
     - Select one or more meshes in Blender.
     - Set the direction switcher in Portal to `Blender ➔ Maya`.
     - Click **ROUNDTRIP SYNC** (or click **Manual Export** in Blender and then **Manual Import** in Maya).

## ⚙ Advanced Options & Policies

Expand the **Advanced Options** drawer in Portal to configure transfer behaviors:

- **Update Existing Mesh by Name**:
  - When enabled, importing an asset updates the existing object's mesh data in place instead of creating duplicate objects (e.g., `Cube.001` or `Cube1`). This preserves existing scene assignments and node networks.
- **Sync Destination Transforms and Pivots**:
  - When enabled, the destination object's position, rotation, scale, and pivot will be stamped with the source object's coordinates.
  - When disabled, only the underlying geometry/UVs update, leaving the destination object's scene placement stable.
- **Open Exchange Folder**:
  - Click **Open Folder** to open your active exchange directory where the raw FBX files and `.bridge_manifest.json` sidecars are exported.

## 🎨 Data & Material Preservation

Portal handles DCC exchange using a **Manifest Sidecar Workflow** to keep asset data intact:

- **Materials**:
  - Existing Blender material slots and shader setups are preserved when a Maya-edited mesh returns. 
  - Duplicate materials generated by the FBX format (e.g., `material.001`) are automatically remapped to their original clean material names.
- **Pivots & Origin**:
  - Custom rotate/scale pivots in Maya and object origins in Blender are tracked and updated accurately using the manifest.
- **Vertex Groups**:
  - Blender vertex groups and weight maps are stored by vertex index and restored after a Maya modeling or UV edit pass (provided the vertex count remains compatible).
- **Sharp/Soft Shading**:
  - Sharp edges marked in the editor are exported via the manifest and correctly shaded using custom attributes and automated viewport modifiers in both hosts.
